## 목표
채팅에서 "메뉴 목록 보여줘"라고 입력하면 AI가 텍스트로 응답하면서 브라우저를 `/menus` 페이지로 이동시킨다.
이 기능은 Gemini Function Calling을 사용하여 구현한다.

## 전체 흐름
```
사용자: "메뉴 목록 보여줘"
  → Agent Server → Gemini에게 대화 전달
  → Gemini가 navigate_to_page(page="menu_list") 함수 호출 결정
  → Agent Server가 함수 실행 → {"action":"navigate","url":"/menus"} 생성
  → Gemini에게 실행 결과 전달 → Gemini가 "메뉴 페이지로 이동할게요!" 텍스트 응답
  → SSE 스트림으로 텍스트 전송 후 액션 JSON 전송
  → 프론트엔드가 액션 감지 → router.push("/menus")로 페이지 이동
```

### 1. `agent-server/app/services/tools.py` — 기존 코드에 추가

navigate_to_page 함수 선언과 실행 로직을 추가한다.

**추가할 내용: 페이지 목록 상수**
```python
PAGES = {
    "home": {"url": "/", "description": "홈페이지"},
    "menu_list": {"url": "/menus", "description": "메뉴 목록 페이지"},
    "login": {"url": "/login", "description": "로그인 페이지"},
}
```

**추가할 내용: AGENT_TOOLS의 function_declarations 배열 안에 추가**
```python
types.FunctionDeclaration(
    name="navigate_to_page",
    description="고객의 브라우저를 특정 페이지로 이동시킵니다. '보여줘', '이동해줘', '가줘' 등 페이지 이동 요청에 사용합니다.",
    parameters_json_schema={
        "type": "object",
        "properties": {
            "page": {
                "type": "string",
                "enum": list(PAGES.keys()),
                "description": "이동할 페이지 (home: 홈페이지, menu_list: 메뉴 목록, login: 로그인)",
            },
        },
        "required": ["page"],
    },
),
```

**추가할 내용: execute_function 안에 navigate_to_page 처리 분기**
```python
if name == "navigate_to_page":
    page_info = PAGES.get(args.get("page", ""))
    if not page_info:
        return {"error": "알 수 없는 페이지"}, None
    action = {"action": "navigate", "url": page_info["url"]}
    return {"success": True, "page": page_info["description"]}, action
```

**핵심:** `execute_function`은 반드시 `(result, action)` 튜플을 반환한다.
- `result` → Gemini에게 전달 (텍스트 응답 생성에 사용)
- `action` → 프론트엔드로 전달 (브라우저 조작에 사용)

아래 두 가지가 되어 있는지만 확인:

```python
config = types.GenerateContentConfig(
    system_instruction=system_prompt,
    tools=[AGENT_TOOLS],  # ← 이것이 있어야 함

```

**(b)** 스트리밍 함수에서 텍스트는 `str`로, 액션은 `dict`로 yield하는가:
```python
# 텍스트 청크 → str로 yield
if chunk.text:
    yield chunk.text

# function call 감지 시 실행 후 action 저장
result, fn_action = await execute_function(fc)
if fn_action:
    action = fn_action

# 텍스트 완료 후 마지막에 action을 dict로 yield
if action:
    yield action
```

### 4. `agent-server/app/routers/chat.py` — SSE 이벤트 생성 확인

스트리밍 event_generator에서 아래 패턴이 되어 있는지 확인:

```python
async for chunk in gemini.chat_stream(history, system_prompt=system_prompt):
    if isinstance(chunk, dict):
        # dict → 프론트엔드 액션 (나중에 전송)
        action = chunk
    elif chunk:
        # str → 텍스트 청크 (즉시 SSE 전송)
        yield {"data": json.dumps({"content": chunk}, ensure_ascii=False)}

# 텍스트 이후에 액션 전송
if action:
    yield {"data": json.dumps(action, ensure_ascii=False)}
yield {"data": "[DONE]"}
```

**SSE로 전송되는 순서:**
```
data: {"content": "메뉴 페이지로 이동할게요!"}   ← 텍스트
data: {"action": "navigate", "url": "/menus"}     ← 액션 (텍스트 뒤에)
data: [DONE]                                       ← 종료 신호
```

### 5. `frontend/app/lib/aiAgent.ts` — SSE 파싱에 액션 처리 추가

기존 SSE 파싱 코드에서 JSON을 파싱하는 부분에 **action 분기만 추가**한다.

**기존에 텍스트만 처리하는 코드가 있다면:**
```typescript
// 변경 전: 텍스트만 처리
const parsed = JSON.parse(data);
if (parsed.content) {
    yield parsed.content;
}
```

**아래처럼 action 분기를 추가:**
```typescript
// 변경 후: 액션도 처리
const parsed = JSON.parse(data);
if (parsed.action) {
    yield { action: parsed };   // 액션 객체로 yield
} else if (parsed.content) {
    yield parsed.content;       // 텍스트 문자열로 yield
}
```

**타입 정의도 추가 (파일 상단):**
```typescript
export type StreamChunk = string | { action: { action: 'navigate'; url: string } };
```

**주의: 기존 sendMessageStream 함수의 반환 타입을 AsyncGenerator<StreamChunk>로 변경**
